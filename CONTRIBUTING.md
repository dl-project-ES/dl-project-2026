# Contributing Guidelines

Thank you for contributing to this project.

## Workflow

- Each team member should work on their assigned tasks in separate branches.
- Avoid directly pushing major changes to the main branch.
- Create pull requests for review before merging.

## Code Structure

- Preprocessing code should be placed in src/preprocessing
- Model-related code should be placed in src/models
- Open-set detection code should be placed in src/open_set
- Clustering code should be placed in src/clustering
- Evaluation scripts should be placed in src/evaluation

## Commit Messages

- Use clear and meaningful commit messages
- Example: "Add signal segmentation script"

## Data Handling

- Do not upload raw or processed datasets to GitHub
- Use local storage or external drives for dataset sharing

## Documentation

- Update README or docs when adding major features
